<objective>
## Goal
Resolve UAT-001 by adding a visible "Description" label to the clarify modal's description field, matching the existing "Title" label pattern.

## Purpose
AC-3 of plan 02-01 made the description field visible and editable, but the field has no header/label. Users encountering an empty textarea during clarification have no indication of what it's for. This fix closes the spec gap (the original AC required visibility but did not require a label).

## Output
- One additional `<label>` element in `app/web_ui/index.html` paired with `#clarifyDescSummary`.
- Visibility of that label kept in sync with the field in `populateClarifyPreview` and `resetClarifyState` (`app/web_ui/js/ui.js`).
- A new `clarifyDescSummaryLabel` entry in `cacheElements()`.

Priority: 0 blocker, 0 major, 1 minor, 0 cosmetic
</objective>
