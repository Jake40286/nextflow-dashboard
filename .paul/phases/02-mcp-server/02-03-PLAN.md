<objective>
## Goal
Refactor the 789-LoC `app/mcp_server.py` monolith into a clean `app/mcp_server/` package, and bundle three small dependency / config cleanups (`task-<uuid>` ID format, `starlette<2` pin, drop obsolete `version: "3.8"` from compose). All MCP tool behavior MUST remain byte-identical from the LLM's perspective — this is a structural refactor, not a behavior change.

## Purpose
The monolith grew 60% in 02-02 (494 → 789 LoC) and is past the comfortable single-file threshold. Splitting it now — *before* the test suite lands in 02-04 — means the test harness targets the final API surface and module boundaries, rather than getting rewritten when the refactor follows. The cleanup items are co-located because each is a one-line change inside files we're already touching, and bundling them avoids a separate trivial loop.

## Output
- `app/mcp_server/` package replacing the single file, with no behavior change.
- `create_task` IDs become `task-<uuid>` (consistent with `data.js generateId('task')`).
- `requirements-mcp.txt` pins `starlette<2`.
- `docker-compose.yml` drops the obsolete `version: "3.8"` top-level key.
- Smoke test: `docker compose up --build -d`, sidecar healthy at `http://127.0.0.1:8003/mcp`, all 10 tools enumerated, one round-trip `create_task` succeeds end-to-end.
</objective>

<boundaries>

## DO NOT CHANGE
- **Tool signatures, docstrings, parameter names, default values** — these are the LLM-facing contract. Any change here belongs in a separate plan with a deliberate decision.
- **Validation logic** in any tool body (status enums, length caps, ID-required guards). Pure relocation only.
- **Optimistic-locking flow** — the 3-retry 409 loop, the `_rev` strip-on-PUT, the completion-field strip. These are the carried-forward Phase 1 decisions.
- **Per-group LWW timestamp logic** in `update_task_status`'s `_set_status` mutator — codified in 02-02.
- `app/web_ui/`, `app/server.py`, `app/google_calendar.py`, `app/backup.py` — main app is not in scope.
- `tests/taskManager.test.js` — JS test suite is unaffected.

## SCOPE LIMITS
- **No new tools.** This plan does not add `complete_task`, edit-note, delete-note, or arbitrary-field-update tools — those are Phase 3 territory.
- **No Python test harness.** That's plan 02-04 by user direction; if a test infra question surfaces here (e.g., "should I add a smoke test?"), defer to 02-04.
- **No auth changes.** Localhost-only bind, no token — locked for Phase 1 PoC, unchanged until Phase 3.
- **No `_source: "mcp"` durability fix.** Carried forward to Phase 3.
- **No `mcp` SDK version bump.** Pinned at 1.27.0; out of scope.
- **No data.js or JS-side changes.** Even if a clean alignment surfaces (e.g., shared status enums), defer.
