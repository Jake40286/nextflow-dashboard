**Two new top-of-page status bars — "My Day" (tasks scheduled or due today) and "Neglected" (top 5 stale active tasks) — mirror the existing urgent-bar pattern, render via `renderAll()`, hide when empty, and open the task flyout on chip click.**

## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Hardcoded hex + `color-mix(... var(--surface))` instead of new `--myday-accent`/`--neglected-accent` variables | Mirrors existing urgent-bar pattern at `style.css:6498` (`#dc2626`); consistency with codebase wins over plan's strict fallback guidance | Theme-adaptive backgrounds via color-mix; pure label colors are not theme-aware. If a future theme refactor introduces accent variables for urgent, both new bars should be migrated together |
| Inbox tasks excluded from Neglected (`status !== 'inbox'`) | Inbox items are unprocessed, not "neglected"; surfacing them in Neglected would conflate two different signals | Neglected is now meaningful as a "slipping work" surface |
| Mirror urgent-bar listener pattern (one click handler per chip) instead of event delegation | Plan boundary protects urgent-bar; introducing delegation only on the new bars would create asymmetry | Acceptable since chip count is bounded by data; refactor to delegation can be done across all three bars in a single future plan |
