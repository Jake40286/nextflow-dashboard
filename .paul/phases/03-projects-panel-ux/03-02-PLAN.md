<objective>
## Goal
Stop flagging a project as "missing next action" when it has at least one task in `STATUS.NEXT`, `STATUS.DOING`, or `STATUS.WAITING`. Today the check only counts `STATUS.NEXT`, so projects whose only active work is delegated (`waiting`) or in-progress (`doing`) are wrongly highlighted with the at-risk styling. Resolves feedback `3ff676c5`.

## Purpose
The "no next action" warning is meant to surface stalled projects ‚Äî ones where the user has captured a project but never decided what's next. A delegated task is implicitly "next" (the next event is the delegate's response), and a `doing` task literally IS the action being taken right now. Treating those projects as at-risk creates noise that erodes trust in the warning's signal.

## Output
- `hasNextAction` map in `panels/projects.js` populated for tasks in NEXT, DOING, or WAITING (was: NEXT only)
- All three downstream consumers (`missingNext` flag at line 35, `showMissingNextOnly` filter at line 39, per-row `missingNext` at line 75) automatically inherit the fix without needing edits
- "Missing next action" toolbar filter (`#toggleMissingNextAction`) continues to work ‚Äî projects with only delegated/doing tasks now correctly disappear from that filtered view
</objective>
