<objective>
## Goal
Three concrete polish items on the Pending Tasks panel and its association filter:
1. Add an "Area of Focus" section to the association filters so users can filter the same way they already filter by Context, Project, etc. (`059f0a1e`)
2. Verify (and fix if needed) that clicking anywhere on a task flyout's Notes or List header expands the section — not just the small toggle button. The code looks like this is already wired; this task confirms it works and surfaces missing visual hover affordance if any. (`f3d948ce`)
3. Two small fixes to the multi-edit bar: stop the selection checkbox from covering task text (cosmetic), and add Effort + Time Required selects so users can bulk-set those fields. Contexts is deliberately deferred to plan 05-03 because it's a multi-value field that fits the bigger redesign there. (`fb700fcc` items 1+2)

## Purpose
These are three of the seven feedback items in Phase 5. They cluster on the same surface — the active task list (Pending Tasks panel) and its surrounding controls — so it's natural to ship them together. None of them have unknowns; the discovery already cleared the design questions, and the bulk-edit redesign work is held back for 05-03.

## Output
- New "Area of Focus" group in the association filter flyout, populated from `taskManager.getAreasOfFocus()`, behaving identically to the existing Context/Effort/Time groups
- Notes and List headers visually communicate clickability (hover state) — and behavior verified working
- Multi-edit bar checkbox no longer overlaps task title text on common card layouts
- Multi-edit bar gains "Set effort…" and "Set time required…" select fields
</objective>
